10 /// CSL Coding Conventions
11 /// Naming
12 /// Class, member, and method names are written in "camelCase" as in "UnitGenerator."
13 /// Class names are capitalized, but members and methods are written lower-camel-case.
14 /// Data members (instance variables) are written with initial "m" then camelCase as in "mOffset."
15 /// Enumeration constants are written with initial "k" followed by embedded caps as in "kDone."
16 /// Compiler flags and #defines are generally written all-upperr-case as in "USE_CASND."
17 /// Privacy
18 /// In general data members are protected and have accessor functions where appropriate.
19 /// The one exception is Buffer which is considered a record class and has public members.
20 ///
21 /// Compile-time options
22 /// (These are not set in this file so that different applications can share the source tree.)
23 ///
24 /// SoundFile type:
25 /// USE_JSND - use the JUCE-based sound file class (requires only JUCE)
26 /// USE_LSND - use libSndFile-based sound file class (supports many file types)
27 /// USE_CASND - use the CoreAudio version (runs on iPhone)
28 ///
29 /// FFT implementation
30 /// USE_FFTW - use FFTW 3 (faster but complicated to build)
31 /// USE_FFTREAL - use FFTReal (smaller and simpler)
32 /// USE_KISSFFT - use KISS FFT (smaller, untested)
33 ///
34 /// Thread implementation
35 /// USE_JTHREADS - use the JUCE thread classes (otherwise use pthreads)
36 ///
37 /// MIDI API
38 /// USE_JMIDI - use JUCE's MIDI I/O
39 ///
40 /// OSC API
41 /// USE_LOSC - use LibLo for OSC (assumes liblio 0.26 or greater is installed)
